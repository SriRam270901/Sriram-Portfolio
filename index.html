Artifact 02

Applied Generative AI

SystemBridge AI: Evidence-Disciplined Technical Triage

Open Live Demo View Documentation

SystemBridge AI is a web-based generative AI assistant designed to turn vague production symptoms into structured, evidence-based investigation plans without pretending to know the root cause.

1Capture the symptom
2Separate facts from assumptions
3Identify missing evidence
4Prioritize low-risk checks
5Create a concise handoff

Objective

Support software developers and technical support engineers during early-stage incident investigation by improving reasoning structure rather than automating production decisions.

Process

Conducted LLM experiments, evaluated prompt sensitivity, applied design thinking, defined behavioral boundaries, built a web prototype, and tested scenarios designed to expose unsafe or unsupported conclusions.

Tools

HTML, CSS, JavaScript, Puter.js, generative AI, GitHub, GitHub Pages, prompt design, scenario testing, and responsible-AI principles.

Value Proposition

Shows end-to-end AI product thinking: problem definition, user-centered design, technical implementation, deployment, privacy boundaries, evaluation, and human-in-the-loop decision support.

Unique Value

Unlike a general chatbot, the assistant is intentionally constrained to distinguish evidence from hypothesis, state uncertainty, avoid secret collection, and reject destructive first actions.

Relevance

Directly supports my goal of building responsible AI applications that fit real enterprise workflows and strengthen, rather than replace, professional judgment.
